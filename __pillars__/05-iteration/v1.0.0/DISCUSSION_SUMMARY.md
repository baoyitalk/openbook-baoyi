## 🎯 项目定位讨论

### 项目类型确认

**讨论时间**: 2025-12-16

**结论**:
- 项目类型：基于 Docusaurus 的开源电子书项目
- 主要用途：知识库 + 技术文档 + 博客系统
- 目标用户：技术学习者、知识分享者

**关键特点**:
1. **内容为王** - 专注于内容质量和组织
2. **静态生成** - 无需后端，部署简单
3. **SEO 友好** - 支持搜索引擎优化
4. **响应式设计** - 完美适配移动端

## 🏗️ 架构设计讨论

### 技术栈选择

**讨论时间**: 2025-12-16

**考虑的方案**:
1. Docusaurus ✅ **最终选择**
2. VuePress
3. GitBook

**选择理由**:

- React 生态，插件丰富

**最终技术栈**:

```
- 框架：Docusaurus v3.9.2
- UI：React 18.3.1
- 包管理：npm
- 部署：静态站点（待定平台）
```

---

### 支柱体系设计

**讨论时间**: 2025-12-16

**核心问题**: 是否需要完整的五支柱体系？

**讨论过程**:
1. 传统软件项目需要五支柱（产品、架构、测试、运维、迭代）
2. 电子书项目主要是内容管理，不涉及复杂业务逻辑
3. 测试支柱对电子书项目意义不大
4. 内容质量通过人工审核，技术质量由 Docusaurus 保证

**最终决策**:
采用**四支柱体系**，不包含测试支柱

**支柱设计**:
```
__pillars__/
├── 01-product/      # 产品支柱 - 内容规划、章节结构
├── 02-architecture/ # 架构支柱 - 技术配置、插件管理
├── 04-operations/   # 运维支柱 - 部署流程、发布管理
└── 05-iteration/    # 迭代支柱 - 版本管理、开发日志
```

**不包含**:
- ~~03-testing/~~ - 测试支柱（电子书项目不需要）

## ⚙️ 配置讨论

### 开发端口配置

**讨论时间**: 2025-12-16

**问题**: Docusaurus 默认使用 3000 端口，可能与其他项目冲突

**解决方案**:
```json
{
  "scripts": {
    "start": "docusaurus start --port 3005",
    "serve": "docusaurus serve --port 3005"
  }
}
```

**访问地址**: http://localhost:3005

---

### 国际化配置

**讨论时间**: 2025-12-16

**决策**: 默认使用中文

**配置**:
```javascript
i18n: {
  defaultLocale: 'zh-Hans',
  locales: ['zh-Hans'],
}
```

**理由**: 目标用户主要是中文用户

---

### 主题配置

**讨论时间**: 2025-12-16

**决策**: 支持深色模式，默认浅色

**配置**:
```javascript
colorMode: {
  defaultMode: 'light',
  disableSwitch: false,
  respectPrefersColorScheme: true,
}
```

**理由**: 提供更好的阅读体验，尊重用户系统偏好

## 📝 开发规范讨论

### Git 分支策略

**讨论时间**: 2025-12-16

**分支模型**:
```
main              # 生产分支，只接受 tag 合并
  └── dev-v1.0.0  # 开发分支，日常开发
```

**工作流程**:
1. 基于 main 创建 dev-vX.X.X
2. 在开发分支上进行所有开发
3. 完成后打 tag（需授权）
4. 合并到 main

**核心规则**:
- ✅ 所有开发必须在开发分支
- ✅ 未经允许不得合并到 main
- ✅ 遵循 Git 提交规范

---

### Git 提交规范

**讨论时间**: 2025-12-16

**规范格式**:
```
feat:     - 新功能
fix:      - 修复 Bug
docs:     - 文档更新
chore:    - 构建/工具/配置更新
refactor: - 代码重构
test:     - 测试相关
style:    - 代码格式调整
```

**特殊规则**:
- 只有在明确允许的情况下，才能使用 `v1.1.xx` 版本号开头命名

## 📝 重要决策记录

### 决策 #1: 不使用测试支柱

**理由**:
- 电子书项目主要是静态内容
- 内容质量通过人工审核
- 技术层面由 Docusaurus 保证
- 避免过度设计

**影响**:
- 文档结构更简洁
- 维护成本更低
- 聚焦内容质量

---

### 决策 #2: 使用 3005 端口

**理由**:
- 避免与其他项目冲突
- 便于同时运行多个项目

**影响**:
- 开发环境访问地址变更
- 文档中需要说明端口

---

### 决策 #3: 中文优先

**理由**:
- 目标用户主要是中文用户
- 简化国际化配置

**影响**:
- 默认中文界面
- 暂不支持多语言
