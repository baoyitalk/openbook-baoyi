## 🚀 部署流程

### 本地构建
```bash
# 安装依赖
npm install

# 本地开发
npm start

# 构建生产版本
npm run build

# 预览生产构建
npm run serve
```

### 部署到生产
```bash
# 1. 构建
npm run build

# 2. 部署（根据托管平台不同）
# - GitHub Pages: npm run deploy
# - Vercel: 自动部署
# - Netlify: 自动部署
```

## 🔧 常见托管平台

### GitHub Pages
- **优点**: 免费、简单、与 GitHub 集成
- **缺点**: 仅支持静态站点
- **部署**: `npm run deploy`

### Vercel
- **优点**: 免费、自动部署、性能优秀
- **缺点**: 国内访问可能较慢
- **部署**: 连接 GitHub 仓库自动部署

### Netlify
- **优点**: 免费、功能丰富、支持表单
- **缺点**: 构建时间有限制
- **部署**: 连接 GitHub 仓库自动部署

### 自建服务器
- **优点**: 完全控制、可定制
- **缺点**: 需要维护、成本较高
- **部署**: 使用 CI/CD 工具

## 📝 最佳实践

### DO ✅
1. ✅ 使用自动化部署，减少人工操作
2. ✅ 配置 HTTPS，保证安全性
3. ✅ 使用 CDN，提升访问速度
4. ✅ 定期备份，防止数据丢失
5. ✅ 监控网站状态，及时发现问题

### DON'T ❌
1. ❌ 不要手动部署，容易出错
2. ❌ 不要忽视 SEO 配置
3. ❌ 不要使用不稳定的托管平台
4. ❌ 不要忽视移动端适配
5. ❌ 不要长期不更新依赖
