这道题确实是经典的「引用类型 + 函数参数传递」混合题，很多人会搞混。让我用图解帮你彻底搞懂。

---

## 🎯 核心概念：函数参数传递的本质

> **JavaScript 函数参数传递永远是「值传递」**，但对于引用类型，传递的是**地址值的副本**。

---

## 📊 逐步图解执行过程

### 第一步：初始状态

```javascript
let person = { name: "John", age: 25 }
```

```
    栈内存                          堆内存
┌─────────────────┐            ┌─────────────────┐
│                 │            │                 │
│  person: 0x001 ─┼───────────▶│ { name: "John", │
│                 │            │   age: 25 }     │
│                 │            │                 │
└─────────────────┘            └─────────────────┘
                                   地址: 0x001
```

---

### 第二步：调用 `increaseAge(person)`

**关键点**：传参时，把 `person` 的值（地址 `0x001`）**复制**给了 `obj`

```
    栈内存                          堆内存
┌─────────────────┐            ┌─────────────────┐
│                 │            │                 │
│  person: 0x001 ─┼─────┬─────▶│ { name: "John", │
│                 │     │      │   age: 25 }     │
│  ─────────────  │     │      │                 │
│  函数作用域      │     │      └─────────────────┘
│  obj: 0x001  ───┼─────┘         地址: 0x001
│                 │
└─────────────────┘

    person 和 obj 指向同一个对象！
```

---

### 第三步：执行 `obj.age += 1`

**通过 obj 修改对象属性**，因为 obj 和 person 指向同一个对象，所以原对象被修改了！

```
    栈内存                          堆内存
┌─────────────────┐            ┌─────────────────┐
│                 │            │                 │
│  person: 0x001 ─┼─────┬─────▶│ { name: "John", │
│                 │     │      │   age: 26 }     │  ← age 变成 26 了！
│  ─────────────  │     │      │                 │
│  函数作用域      │     │      └─────────────────┘
│  obj: 0x001  ───┼─────┘         地址: 0x001
│                 │
└─────────────────┘
```

---

### 第四步：执行 `obj = { name: "Jame", age: 22 }`

**关键点**：这里是给 `obj` **重新赋值**，让它指向一个**新对象**！

```
    栈内存                          堆内存
┌─────────────────┐            ┌─────────────────┐
│                 │            │                 │
│  person: 0x001 ─┼───────────▶│ { name: "John", │
│                 │            │   age: 26 }     │
│  ─────────────  │            └─────────────────┘
│  函数作用域      │               地址: 0x001
│  obj: 0x002  ───┼──┐
│                 │  │         ┌─────────────────┐
└─────────────────┘  │         │                 │
                     └────────▶│ { name: "Jame", │
                               │   age: 22 }     │  ← 新对象！
                               └─────────────────┘
                                  地址: 0x002

    现在 obj 和 person 指向不同的对象了！
```

---

### 第五步：`console.log(obj)` // 输出 1

```javascript
// 输出: { name: "Jame", age: 22 }
// 因为 obj 现在指向新对象 0x002
```

---

### 第六步：函数执行完毕，`console.log(person)` // 输出 2

函数作用域销毁，`obj` 变量消失，新对象 `0x002` 没有引用会被垃圾回收

```
    栈内存                          堆内存
┌─────────────────┐            ┌─────────────────┐
│                 │            │                 │
│  person: 0x001 ─┼───────────▶│ { name: "John", │
│                 │            │   age: 26 }     │  ← 还是这个对象
│                 │            │                 │
└─────────────────┘            └─────────────────┘

// 输出: { name: "John", age: 26 }
```

---

## ✅ 最终答案

```javascript
console.log(obj);    // 1: { name: "Jame", age: 22 }
console.log(person); // 2: { name: "John", age: 26 }
```

---

## 🔑 核心要点总结

```
┌─────────────────────────────────────────────────────────────────┐
│                      两种操作的本质区别                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   obj.age += 1          vs         obj = { ... }               │
│        │                                 │                      │
│        ▼                                 ▼                      │
│   通过地址修改                       修改 obj 变量本身           │
│   堆内存中的对象                     让它指向新地址              │
│        │                                 │                      │
│        ▼                                 ▼                      │
│   ✓ 会影响原对象                    ✗ 不影响原对象               │
│   (person 也能看到变化)             (person 还指向原来的)        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

---

## 📝 记忆口诀

> **「改属性，动真格；换地址，各走各」**
>
> - `obj.xxx = yyy` → 修改的是对象内容，原变量会受影响
> - `obj = yyy` → 修改的是 obj 这个变量本身，原变量不受影响

这道题的陷阱就在于：**先通过 obj 改了属性（生效），后给 obj 换了新地址（不影响 person）**。
